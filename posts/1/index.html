GitHub Pages 초기 설정

December 15, 2020

1. 서론

블로그는 서비스형과 설치형이 있습니다.
서비스형인 싸이월드 블로그와 네이버 블로그를 이용하면서 내가 작성한 컨텐츠 원본을 내 디스크에 저장하지 못한다는 것이 아쉬워 설치형 워드프레스로 운영했던 적이 있습니다. 내 서버에 내가 생성한 컨텐츠가 저장되니 온전히 내것이었죠.
하지만, 여러 번의 이사, 컴퓨터 교체, 워드프레스 및 플러그인의 잦은 업데이트로 인해 항상 관리해주어야 한다는 부담감과 귀찮음에 운영을 중단했고, 이후 블로그는 아니지만 페이스북에 한동안 빠졌다가 글 작성을 완전히 멈췄습니다.
그래도 개발 관련 사항을 정리해야겠다는 생각은 항상 가지고 있어서 GitHub Pages 서비스 오픈 시기부터 눈여겨 보다가 이제서야 이용하게 되었습니다. 내 Git 저장소에 저장되어 온전히 내것이고, 설치형이 아니라서 관리의 귀찮음도 없는 너무 편리한 서비스입니다.

.

2. GitHub Pages

  • 참고: https://pages.github.com

2-1. Git 저장소 생성

https://github.com에 가입되었다면 Repositories - New를 선택해 저장소를 생성합니다.

  • Repository name: 아이디/github.io
  • Public 터미널을 이용해 Git 명령어로 간단하게 사이트를 만들 수 있습니다.
  • 명령어 중 아이디 부분을 자신의 GitHub 아이디로 교체해 주세요.
# Git 저장소 복제
$ git clone https://github.com/아이디/아이디.github.io

# 테스트 HTML 작성
$ cd 아이디.github.io
$ echo "Hello World" > index.html

# Git 푸시
$ git add -all
$ git commit -m "Initial commit"
$ push -u origin main

이제 웹 브라우저로 https://아이디.github.io 에 접속해 보면 Htllo World가 출력될 것입니다.
