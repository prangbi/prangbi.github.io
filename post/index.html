Go 설치 및 VSCode 설정

1. Go 설치 참고: https://golang.org/doc/install 맥과 윈도우에서는 설치파일을 다운받아 설치하면 됩니다. 리눅스에서는 실행파일이 압축파일로 되어 있어 특정 디렉토리에 풀고 PATH를 설정해 주면 됩니다. ## 실행파일을 다운받아 /usr/local에 압축해제 하면 /usr/local/go 디렉토리가 생성됩니다. $ sudo tar -C /usr/local -xzf https://golang.org/dl/go1.15.6.linux-amd64.tar.gz ## Go 실행 PATH 추가 $ echo 'PATH="$PATH:/usr/local/go/bin"' >> ~/.profile ## .profile 설정 재로딩 $ source .profile Go 설치 확인 ## Go 실행 확인 $ go version go version go1.

Visual Studio Code (VSCode)

1. 서론 (Visual Studio Code = VSCode) 1-1. VSCode 사용 전 개발툴은 다양합니다. 저는 툴을 잘 다루지 못해서, 타겟 플랫폼 또는 언어에 최적화 되어 있고 추가 설정 필요 없는 IDE가 좋습니다. 하지만, Go 언어 개발용 LiteIDE를 제외하고는 점점 무거워지고, Mac/Linux/Windows 사용 중이라 OS에 따라 성능 차이가 심하게 나는 점 때문에 다른 툴이 없나 계속 찾아보곤 했죠. 1-2. VSCode 사용 계기 그러다 GitHub에서 만든 Atom 에디터를 접했는데, 단순 에디터로 사용하기 편한 UI로 되어 있어 간단한 텍스트 편집용으로 종종 사용해 보다가, 느린 속도와 개발용으로는 별도 설정을 해주어야 해서 사용하지 않았습니다.

GitHub Pages 초기 설정

1. 서론 블로그는 서비스형과 설치형이 있습니다. 서비스형인 싸이월드 블로그와 네이버 블로그를 이용하면서 내가 작성한 컨텐츠 원본을 내 디스크에 저장하지 못한다는 것이 아쉬워 설치형 워드프레스로 운영했던 적이 있습니다. 내 서버에 내가 생성한 컨텐츠가 저장되니 온전히 내것이었죠. 하지만, 여러 번의 이사, 컴퓨터 교체, 워드프레스 및 플러그인의 잦은 업데이트로 인해 항상 관리해주어야 한다는 부담감과 귀찮음에 운영을 중단했고, 이후 블로그는 아니지만 페이스북에 한동안 빠졌다가 글 작성을 완전히 멈췄습니다. 그래도 개발 관련 사항을 정리해야겠다는 생각은 항상 가지고 있어서 GitHub Pages 서비스 오픈 시기부터 눈여겨 보다가 이제서야 이용하게 되었습니다.
